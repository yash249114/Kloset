## Findings & Fixes

### 1. FONT-SANS CSS VARIABLE MISMATCH (Fixed)

**File:** `globals.css:24`
**Severity:** High

The Tailwind `@theme` block declared `--font-sans: "Inter"` but the body CSS rule applied `'DM Sans'`. This meant Tailwind's `font-sans` utility resolved to Inter while the body used DM Sans — a silent type ramp conflict.

**Fix:** Changed `--font-sans: "Inter"` to `--font-sans: "DM Sans"` to align with the Google Fonts import and body rule.

### 2. NAVBAR/PAGE SPACING MISMATCH (Fixed)

**File:** `page.tsx:89`
**Severity:** Medium

Homepage applied `pt-[72px]` to offset the navbar, but the navbar is `h-[64px]`. This created an 8px gap of ivory background visible above the hero.

**Fix:** Changed `pt-[72px]` to `pt-[64px]`.

### 3. SECTION BACKGROUND INCONSISTENCY (Fixed)

**Files:** `Trust.tsx`, `HowItWorks.tsx`, `loading.tsx`
**Severity:** Medium

Trust and HowItWorks used `bg-white` (#ffffff) while Categories, Reviews, and the rest used `bg-ivory` (#FAF7F2). Two consecutive white sections broke the visual rhythm. The Categories skeleton also used `bg-white` while the actual section used `bg-ivory`.

**Fix:** Unified all light sections to `bg-ivory`. Skeleton matches its section.

### 4. MOBILE NAV BRAND INCONSISTENCY (Fixed)

**File:** `MobileNavDrawer.tsx:89`
**Severity:** Medium

Mobile drawer displayed plain text "KLOSET" in Playfair Display while the desktop navbar used the SVG KlosetMonogram + KlosetWordmark. This was a visible brand inconsistency on mobile.

**Fix:** Replaced plain text with `KlosetMonogram` + `KlosetWordmark` SVG components.

### 5. REVIEWS CARD HOVER DUPLICATION (Fixed)

**File:** `Reviews.tsx:51`
**Severity:** Low

Review cards used inline hover styles (`hover:translate-y-[-4px] hover:shadow-xl hover:border-champagne/20`) duplicating the existing `hover-lift` utility class defined in globals.css.

**Fix:** Replaced inline styles with `hover-lift` class.

### 6. HERO AI STYLIST BUTTON MISSING VARIANT (Fixed)

**File:** `Hero.tsx:75`
**Severity:** Low

The AI Stylist button used `btn` without a variant class, applying inline border/color styles manually. Other ghost-style buttons across the app use `btn btn-ghost`.

**Fix:** Added `btn-ghost` variant class.

### 7. SELLER CTA IMAGE OVERLAY (Fixed)

**File:** `SellerCTA.tsx:84`
**Severity:** Low

Image overlay used `bg-gradient-to-l` (left-justified) which created an asymmetric gradient that didn't match the Hero's `bg-gradient-to-r` pattern. Also missing `relative` on the container for the absolute-positioned gradient.

**Fix:** Changed to `bg-gradient-to-t` for subtle upward fade. Added `relative` to container.

### 8. GRAIN OVERLAY Z-INDEX (Fixed)

**File:** `globals.css:192`
**Severity:** Low

Grain texture overlay used `z-index: 9999` which could interfere with modals, drawers, and toast notifications.

**Fix:** Reduced to `z-index: 50` — above content but below interactive overlays.

## Animation Audit

All animations use the shared `easeConfig` (0.8s, cubic-bezier 0.16, 1, 0.3, 1):
- **Hero:** Background scale 1.06→1 (1.8s), staggered text fade-up
- **Sections:** Fade-up on scroll (whileInView, once)
- **Cards:** Hover lift via CSS transitions
- **Mobile drawer:** Spring slide from left

No heavy keyframe animations. No infinite loops. No parallax. Meets "minimal" requirement.
