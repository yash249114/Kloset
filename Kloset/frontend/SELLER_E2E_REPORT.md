## Issues Found & Fixed

### 1. TypeScript Build Error — `Trending.tsx:60`
- **File:** `components/home/Trending.tsx`
- **Error:** `Type 'string | undefined' is not assignable to type 'string'` on `alt={item.title}`
- **Fix:** Changed to `alt={item.title || 'Trending outfit'}`
- **Root cause:** Component accepts `Partial<Outfit>[]`, making `title` optional.

### 2. Analytics Duplicate Data Load — `analytics/page.tsx`
- **Issue:** `useEffect` duplicated the `loadData()` logic inline via a local `init()` function, while `loadData` was only used by "Sync Analytics" button.
- **Fix:** Replaced the duplicate inline logic with `loadData()` call from `useEffect`.

### 3. Inbox Search Not Wired — `inbox/page.tsx`
- **Issue:** Search `<input>` had no `onChange` handler — decorative only.
- **Fix:** Added `searchTerm` state, `onChange={(e) => setSearchTerm(e.target.value)}`, `filteredConversations` memoization, and used `filteredConversations` instead of `conversations` in the list render. Added empty state for no search results.

### 4. Support Ticket Field Names — `support/page.tsx`
- **Observation:** Uses `renterName`/`renterEmail` keys (matches `TicketPayload` type and backend API contract). Not a bug — the API expects these field names. No change needed.

- `/seller/earnings` — fetches 10 at once (no pagination)
